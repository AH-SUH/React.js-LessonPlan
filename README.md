# Personal Dashboard (React Learning Project)

## Overview

This project is a guided React learning experience where a single application is built progressively alongside course lessons.

Rather than completing isolated exercises, each concept is applied directly to a **Personal Dashboard app** that evolves over time. This approach reinforces learning through real-world application and continuous development.

---

## Project Goal

To build a functional React dashboard while demonstrating understanding of:

- JSX
- Components
- Props & State
- Event Handling
- Conditional Rendering
- Lists & Data Mapping
- Hooks (`useState`, `useEffect`)
- Routing
- Context API
- Performance & Best Practices

---

## Learning Approach

Each lesson introduces a concept that is immediately applied to the same application.

This approach emphasizes:
- Continuous project growth
- Reinforcement of concepts through application
- Real-world development practices

---

## Project Structure (Lesson Breakdown)

### Lesson 1: Setup & Introduction
- Initialize React app using Vite
- Run development server
- Replace default starter content

**App State:**  
Basic dashboard with heading and introductory text

---

### Lesson 2: JSX & Layout
- Build UI structure using JSX
- Add sections (header, content areas)

**App State:**  
Static dashboard layout

---

### Lesson 3: Components
- Create reusable components (e.g., `TaskBoard`, `ProfileCard`)
- Organize UI into separate files

**App State:**  
Component-based layout structure

---

### Lesson 4: State & Props
- Introduce `useState`
- Store and pass data between components

**App State:**  
Dynamic data rendered in the UI

---

### Lesson 5: Events & Forms
- Handle user input
- Add form for creating new tasks

**App State:**  
User-generated tasks added dynamically

---

### Lesson 6: Lists & Conditional Rendering
- Render lists using `.map()`
- Add conditional UI (e.g., completed tasks)

**App State:**  
Interactive task list with toggle functionality

---

### Lesson 7: Hooks & Lifecycle
- Use `useEffect`
- Handle side effects (logging, simulated data loading)

**App State:**  
Reactive behavior based on state changes

---

### Lesson 8: Routing
- Implement navigation between pages
- Create multiple views (Dashboard, Profile, etc.)

**App State:**  
Multi-page application with navigation

---

### Lesson 9: Context API
- Introduce global state management
- Reduce prop drilling

**App State:**  
Shared data across components

---

### Lesson 10: Performance & Best Practices
- Refactor code
- Improve structure and readability

**App State:**  
Clean, maintainable application
